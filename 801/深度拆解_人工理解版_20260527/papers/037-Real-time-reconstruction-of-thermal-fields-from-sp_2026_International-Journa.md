## 1. 身份层

这是一篇面向高超声速飞行器在线热监测的机器学习反问题方法论文。它的真实身份不是普通传感器插值，也不是单纯把 TFNO 用到热场，而是构建一个“稀疏传感器 + 物理先验 + 神经算子”的实时全场热流/温度重构框架。研究对象包括两类热场：Mach 10.6 钝锥表面热流分布，以及辐射加热 C/SiC 复合材料板的瞬态三维温度场。

方法组合包括 TFNO、replication-masking 数据增强、低保真 CFD/RF 先验、飞行状态输入、多通道张量构造、历史测量窗口、Tucker 张量分解、与插值/BPNN/POD/FNO/UNO/Co-Kriging/Kriging 等 baseline 对比。主要证据来自两个算例、多组消融、遮蔽传感器测试、低保真质量阈值分析、不同算子骨干对比和实时推理延迟。目标读者是逆热传导、热防护在线监测、稀疏传感器重构、神经算子和高超声速热管理研究者。

材料边界：本篇 `metadata.json`、`abstract.txt`、`clean_body.md`、`figure_captions.txt`、`table_captions.txt`、`tables/*.md`、`equations.txt`、`references.txt` 均已读取。抽取质量总体较好但噪声不少：abstract 后混入缩略语、Nomenclature、作者邮箱和 DOI；metadata 作者字段为空；图题中 Fig. 4.7 的说明重复且插入正文式长段；正文中 Table 4.5/Table 5.3 标题被误作章节；表格文件实际到 `table_016.md`，而正文 Omitted Tables 只列到 Table 13；`table_007.md` 有列错位；references 末尾混入结论、CRediT、基金和竞争声明。数值主线可用，但表号和参考文献需 PDF 核查。

## 2. 主张层

一句话主张是：从稀疏且可能任意布置的热传感器实时重构完整热场，可以用 TFNO 作为全局频域算子骨干，并通过 replication-masking、状态/低保真 CFD 融合和历史测量窗口分别解决传感器布局泛化、空间物理先验不足和瞬态时间一致性问题，从而在 0.08 s 级推理内达到低于 2% 的热流误差和低于 1.5% 的温度误差。

主张分四层。第一，实时全场重构主张：不是只估计传感器点或边界参数，而是从 9-126 个传感器恢复 2400-3700 节点场。第二，稀疏布局泛化主张：RM 随机遮蔽并复制训练样本，使模型能面对未见过的传感器布局和传感器失效。第三，多源融合主张：热流重构中离散测量给局部硬约束，飞行状态给全局幅值/形状标定，CFD 低保真场给空间平滑先验。第四，瞬态重构主张：C/SiC 板温度场中，3 个相隔 25 s 的历史测量显著提升时间连续性和空间平滑性。

摘要完整意译为：高超声速飞行器热流和温度实时监测对安全、控制和热管理很关键，但飞行中传感器稀疏，尤其前缘等极端区域难以布置传感器，导致空间信息不完整；作者提出基于 TFNO 的增强方法，从稀疏测量重构完整热场；首先用 replication-masking 随机遮蔽重复样本来扩展训练集；其次在有状态信息时，将状态和低保真数据一起输入模型以提高重构；再次对时间连续重构加入近期测量历史以利用时间相干性；在 Mach 10.6 钝锥热流和 C/SiC 复合板辐射加热温度两个任务中，仅用 9-126 个传感器即可重构 2400-3700 节点场，平均相对误差低于 2%，平均推理延迟约 0.08 s。

结论完整意译为：本文建立基于 TFNO 的物理引导数据驱动反演框架，从稀疏测量重构高超声速构型完整热场；该方法集成 RM 以适应任意稀疏传感器布局，在钝锥气动加热任务中融合离散传感器、飞行状态和低保真 CFD，在 C/SiC 板瞬态任务中引入短历史测量；两个场景均表明方法可实时推理，并优于传统插值、常规神经网络和其他神经算子骨干；钝锥融合三类输入后平均相对误差低于 2%，消融确认各组件贡献；C/SiC 板用 3 个相隔 25 s 的历史测量把误差降到低于 1.5%，无历史时约 6.4%；单场推理约 0.08 s，即使部分传感器失效或位置未测也能保持较好准确性。

主张闭合度较强，但“physics-guided”需要谨慎理解：物理性主要来自低保真场、坐标通道、谱低通和平滑先验，不是显式 PDE 约束或物理残差。

## 3. 选题层

研究问题可以写成：飞行器实际只能布置少量离散热传感器时，如何在任意传感器布局、有限实验样本和实时约束下，稳定重构完整热流或温度场？

选题价值来自三个工程矛盾。第一，热管理需要全场热状态，但飞行中红外热成像和温敏漆难以部署，热电偶/热流传感器只能稀疏布置。第二，最危险区域如前缘、烧蚀区或高热流区往往最难布置传感器，使“已有传感器点准确”不等于“关键区域可知”。第三，传统反问题优化和物理约束方法可解释但慢，纯数据驱动方法快但在小样本和传感器布局变化时容易过拟合。

作者把问题压缩为三个可解决子问题：稀疏输入如何泛化到未见布局，局部传感器如何补足全局空间结构，瞬态序列如何保持时间相干。每个子问题都对应一个方法模块，而不是只换一个更大的网络。这使选题具备方法论文的清晰度。

研究边界也要明确。钝锥热流高保真标签来自历史风洞实验的稳态热流推断，低保真来自 CFD 和工程算法；C/SiC 温度场来自 18 个热电偶的辐射加热实验，缺少真实全场实验标签时，重构准确性的验证主要依赖已知/遮蔽传感器点。方法声称实时推理，但训练成本很高，C/SiC 复制次数增加时单 epoch 已达到数百秒。

## 4. 位置层

文献版图被作者分成三类。第一是优化型反演，通过调整边界条件或参数反复调用前向模型匹配测点，优点是物理一致，缺点是难实时。第二是数据驱动反演，离线训练测量到场的映射，在线快，但离线数据成本高且外推弱。第三是 physics-aware 方法，包括 physics-guided、physics-informed、physics-encoded，可降低数据需求，但常受训练慢、架构敏感、泛化有限约束。

作者制造 gap 的方式是先承认逆热传导领域已有丰富方法，再指出现有方法难同时满足实时、高分辨率、有限数据泛化和任意传感器布局；许多研究还停在边界参数、传感器位置值或简化边界形式，而不是完整空间场。这个 gap 基本成立，尤其对“任意稀疏布局 + 高分辨率场 + 实时”这个组合成立。

最接近工作包括 Tikhonov/共轭梯度/优化反演、ANN-GA 替代前向模型、BPNN/CNN/LSTM/GAN 数据反演、PINN/ensemble PINN/热导率联合识别，以及 DeepONet/FNO/TFNO 类神经算子。本文接在这些工作之后，把 neural operator 的函数空间映射能力用于稀疏传感器场重构，并通过 RM 与低保真融合补足小样本问题。

引用策略较完整：引言按方法类别归纳文献，方法段引用 RF、FNO、TFNO、RM 数据增强，算例段引用 NASA 钝锥实验、工程气动热算法、石英灯热流计算和神经算子 baseline。风险是文献中年份/引用格式有抽取错误，如 `Faroughi et al., 2211`、`Kossaifi et al., 2310`、`Rahman et al., 2204`，需要 PDF 核正。

## 5. 贡献层

作者声明三项贡献：TFNO+RM 泛化稀疏传感器布局；状态、低保真场和传感器的多保真融合降低过拟合和非物理振荡；历史测量输入改善瞬态重构时间一致性。真实贡献可以分为四层。

第一是问题重构贡献：把稀疏热监测从“插值少量测点”提升为“受物理先验约束的全场函数重构”。第二是训练策略贡献：RM 通过复制样本并随机遮蔽传感器，使模型在训练中经历多种观测缺失模式。第三是信息分工贡献：作者通过消融明确了测量、状态、CFD 低保真和工程算法低保真分别带来什么，尤其指出低保真数据质量不足会负迁移。第四是泛化验证贡献：同一框架覆盖外流气动热流与固体复合板瞬态温度两个差异很大的任务。

创新强度属于中等偏强的应用方法整合。TFNO、FNO、RM、RF 和多保真融合都不是全新组件，但它们被放到高超声速热监测的稀疏传感器场重构问题中，并用消融、低保真质量阈值和两类算例支撑，贡献比“换模型”更完整。

贡献边界也很重要。钝锥案例只有 40 个实验工况，测试为 3 个 hold-out 条件，存在训练包络依赖；C/SiC 案例的遮蔽验证仍在同一实验时序和同一固定传感器布局内；全场真实误差无法完全由稀疏测点验证；低保真 CFD 的可用性和质量是强前提；工程算法先验会负迁移，说明“多源融合”不是无条件有效。

## 6. 方法层

方法流程可以归纳为：

```text
构建低保真模型或低保真场 -> 整理高保真稀疏传感器数据
-> 复制样本并随机遮蔽传感器 -> 构造多通道稠密网格输入
-> 坐标/状态/低保真/测量/历史/掩码共同输入 TFNO
-> 训练完整热场输出 -> 在线阶段冻结模型做 0.08 s 级前向推理
```

关键输入通道包括坐标通道、状态通道、低保真场通道、当前或历史测量通道、mask 通道。mask 通道的作用很关键，它让模型区分“无测量填零”和“物理值为零”。TFNO 的频域全局卷积负责捕捉长程空间相关，谱模态截断提供低通平滑先验，Tucker 分解压缩频域权重以降低参数和过拟合风险。RM 的作用是把固定传感器数据变成多种缺失布局训练样本。

钝锥任务的高保真数据来自 Cleary Mach 10.6 风洞实验，三种鼻半径、角攻角 0-20°、单位雷诺数三个水平，共 40 个实验案例，其中 3 个测试。低保真 CFD 用 Fluent RANS/Transition SST，压力 25-175 Pa、攻角 0-20° 采样，三种鼻半径得到 432 个 CFD 样本；工程算法低保真也被生成用于对比。TFNO 输出 100×37 的极坐标网格热流场，最优超参数包括 streamwise 模态 6、azimuthal 模态 8、Fourier 层 4、GELU 和 Tucker 保留 10%。

C/SiC 板任务使用 400×200×3 mm 复合板、6 灯石英加热、18 个 K 型热电偶、0.5 s 采样、3894 个时刻。输出网格为 40×20×3，最优历史设置为 3 个历史测量、25 s 间隔；由于传感器固定且空白区域大，模型采用更激进低通截断和更高 Fourier block 神经元数。

关键假设包括：训练数据覆盖在线状态范围；CFD 低保真趋势与真实热流一致；传感器点能代表主要空间模式；谱低通不会抹掉关键局部峰值；历史窗口足以表示瞬态趋势；遮蔽测试能代表传感器失效或未知位置需求。

## 8. 结构语言层

论文结构是“反问题文献版图 -> 问题数学化 -> 方法三模块 -> 钝锥热流验证 -> C/SiC 温度验证 -> 结论”。引言非常标准地按 optimization-based、data-driven、physics-aware 三类综述，再收束到实时、高分辨率、有限数据和任意传感器布局四个要求。Section 2 用公式把问题从 sensor-to-field 扩展到 multi-source fusion 和 history-augmented reconstruction。Section 3 先讲数据流程，再讲 TFNO 数学与架构。Sections 4/5 分别承担外流热流和结构温度两个场景的证据。

段落功能上，钝锥结果部分先做超参数和数据增强选择，再展示完整重构，再做输入源消融，再分析低保真质量阈值，最后与 baseline 和不同输出形式/算子骨干对比。C/SiC 部分则先说明实验和训练配置，再展示空间场，再用已知点/遮蔽点时间序列和历史窗口消融证明瞬态能力。这种结构把“方法有效”拆成准确性、平滑性、泛化性、实时性和鲁棒性。

语言画像上，高频词包括 `sparse measurements`、`full thermal field`、`reconstruction`、`low-fidelity`、`flight states`、`history`、`masking`、`smoothness`、`real-time`。这些词分别服务问题、输入、方法和性能主张。句式常见 “To mitigate... we introduce...”“This indicates that...”“Compared with...”，属于强实验消融驱动语言。主动语态用于贡献声明，方法与结果多用被动语态，呈现工程客观性。

本文语言的亮点是频繁解释每个输入源的功能分工，而不是只报误差。例如“discrete measurements provide local hard constraints; flight states offer cross-condition scaling; CFD fields supply spatial priors”这类表达很适合复用。风险是部分句子把平滑性和物理性绑定较紧，若没有显式物理方程约束，最好写成“physically plausible prior”而非强物理一致性。

## 9. 引用风险层

引用功能分为六组：全场测量困难和逆热传导背景；优化型反演和正则化；数据驱动/深度学习反演；physics-aware/PINN 方法；神经算子、FNO/TFNO/UNO 与 RF/RM；钝锥 NASA 实验、工程气动热算法和 C/SiC 辐射加热实验相关文献。整体引用密度足够，能支撑方法版图。

审稿风险有八处。第一，physics-guided 风险：缺少 PDE residual 或守恒约束，物理引导主要来自低保真场和频域低通，措辞应避免等同 PINN。第二，低保真依赖风险：最强热流结果依赖 CFD 先验，若 CFD 本身成本高或偏差大，在线链路优势会下降。第三，数据规模风险：钝锥高保真实验仅 40 例，hold-out 仅 3 例，状态外推能力仍有限。第四，验证标签风险：C/SiC 全场重构主要用稀疏热电偶遮蔽验证，不是真正全场温度真值。第五，传感器布局风险：RM 模拟任意布局，但真实布局受结构、安全和热区约束，不一定等同随机遮蔽。第六，baseline 公平性风险：BPNN 在点误差上很强，TFNO 的优势需强调全局场而不是单点拟合。第七，实时性风险：0.08 s 只算已训练模型前向推理，低保真生成和模型重训不在内。第八，引用抽取风险：多个 arXiv 年份和参考文献格式被 OCR 错读，正式引用需核对。

## 10. 复用层

可复用选题角度是：面对稀疏测量反问题，不要只问“用什么模型更准”，而要拆成“布局泛化、空间先验、时间相干、低保真质量、实时推理”五个子问题，并给每个子问题一个可验证模块。

可复用 gap 表达是：Existing inverse heat-transfer methods can be accurate, fast, or physics-aware in isolation, but onboard thermal management requires the simultaneous reconstruction of high-resolution fields from sparse and changing sensor layouts under real-time constraints.

可复用贡献表达是：The contribution is a modular reconstruction framework in which masking improves sensor-layout robustness, low-fidelity fields provide spatial priors, state variables calibrate global conditions, and neural-operator layers enforce coherent field-level mappings.

可复用论证链是：

```text
飞行在线热监测需要全场 -> 传感器稀疏且关键区难布置
-> 传统反演慢，纯数据驱动易过拟合 -> 神经算子提供全局场映射
-> RM 处理布局缺失 -> 状态和低保真场补物理先验
```

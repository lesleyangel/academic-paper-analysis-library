## 1. 身份层

这是一篇面向再入制导的在线优化制导算法论文。它的真实身份不是单纯事件触发 MPC，也不是单纯参数估计，而是把事件触发非线性模型预测制导（ET-NMPG）、在线轨迹优化、权重自适应参数估计（WAPE）和飞行中参数突变检测组合成一个再入制导框架，用来在模型偏差和扰动下减少制导更新次数、提高终端精度并满足再入路径约束。

研究对象是翼身/有升力再入飞行器的无动力滑翔再入过程。状态包括半径、经纬度、速度、航迹倾角、航向角等，控制量是攻角和倾侧角。模型偏差包括大气密度参数、升阻系数和质量等；约束包括动压、热流和过载。方法组合包括伪谱/轨迹优化定义的 NMPG、事件触发条件、在线 WAPE、变点检测、FC-NMPG/ET-NMPG/FLG 对比、Monte Carlo 和飞行中故障仿真。

本报告读取了全部指定文件。metadata 作者为空；table_captions 只有编号；tables 共 6 个，覆盖算法参数、初末状态、模型参数偏差、三种方法统计、终端误差和故障分组统计；figure captions 较完整但中文/数学符号抽取为问号较多；references.txt 编号不稳定；equations.txt 有 35 处 `needs_review`，公式多且符号抽取噪声较大，但运动方程、事件触发、参数估计和权重公式可辨认。需 PDF 复核作者、公式和图中符号。

## 2. 主张层

一句话主张是：**在再入 NMPG 中，用状态偏差触发轨迹优化可以降低不必要在线计算，而在事件触发时引入 WAPE 在线修正大气/气动/质量参数并用变点检测避免数据污染，可以进一步减少制导频率、提高成功率和终端精度，并在路径约束附近保持飞行安全。**

摘要完整意译为：本文提出由权重自适应参数估计辅助的事件触发模型预测制导方法并用于再入制导；基于在线轨迹优化的制导方法在复杂问题中面临制导精度和制导指令计算效率的权衡；用状态偏差超过阈值作为事件触发条件，可在保证一定精度的同时减少计算资源消耗；但实际模型参数常偏离参考值，导致事件触发频率过高，甚至使轨迹优化不可行；为解决该问题，本文在通用 ET-NMPG 框架中用 WAPE 在线更新模型参数，提高制导精度并降低制导频率；对再入过程而言，该方法还能更好保证飞行状态位于可接受路径约束内；同时设计变点检测避免飞行中参数突变导致的数据污染；数值仿真确认有效性。

结论完整意译为：本文提出 WAPE-assisted ET-NMPG 再入制导方法。不同于大多数时间触发的在线优化制导，ET-NMPG 通过事件触发机制在制导精度和计算资源之间取得更好平衡；在此基础上加入参数估计步骤，提高模型预测精度，进一步减少制导周期数量、提升制导精度并保证飞行安全；仿真表明 WAPE-assisted ET-NMPG 相比 UI-NMPG、FC-NMPG 和 ET-NMPG 更有应用潜力。未来需要考虑实际飞行中的更多偏差源和传感器数据，并深入研究事件触发阈值选择和是否引入鲁棒 NMPG。

主张分四层：事件触发优于固定周期；WAPE 能减少模型偏差；WAPE-assisted ET-NMPG 提高 Monte Carlo 成功率和终端精度；变点检测能处理飞行中参数突变。证据链覆盖这些 claim，但 UI-NMPG 没有实际仿真，仅在框架层比较。

## 3. 选题层

再入制导的核心困难是：无动力滑翔飞行器控制能力有限，热流、动压和过载路径约束严格，初始状态、环境和气动参数又存在偏差。传统“轨迹设计 + 跟踪制导”需要在参考轨迹里预留冗余，且欠驱动系统只能跟踪少数状态，导致终端散布较大。

在线轨迹优化型 NMPG 有潜力直接根据当前状态重算指导律，但计算负担大。UI-NMPG 过度频繁，FC-NMPG 固定周期又可能在接近约束时更新不及时。事件触发提供折中：只有状态偏差超过阈值才求解新 TOP。但如果模型参数偏差大，事件会频繁触发，甚至轨迹优化失败。

本文选题的价值就在这个二阶矛盾：不是单纯减少制导更新，而是在减少更新的同时通过在线参数估计让模型更接近真实飞行，使后续事件触发变少、路径约束更安全。研究边界是数值仿真再入问题，使用假定可获得的大气密度、加速度和飞行状态数据；不是飞行试验验证。

## 4. 位置层

文献版图包括四条线。第一条是常规再入制导和轨迹跟踪方法，说明传统方法存在冗余和状态散布问题。第二条是计算制导与在线轨迹优化，包括伪谱法、序列凸优化、MPSP、NMPG 等，说明在线优化有潜力但效率是瓶颈。第三条是 MPC/事件触发控制，提供“按需更新”的概念来源。第四条是在线模型识别和参数估计，说明实际飞行中模型参数偏差需要被在线修正。

Introduction 的 gap 是：现有 NMPG 策略难以同时处理计算效率、路径约束和模型参数偏差；ET-NMPG 可减少计算但在参数偏差下触发频率高、求解失败；在线参数估计虽重要，但直接最小二乘受数据有限、噪声和气动非线性影响，可能给出不合理估计。本文接在 ET-NMPG 和在线参数估计之间，提出 WAPE 辅助框架。

这个 gap 成立，尤其是“事件触发频率受模型偏差驱动”的问题具有工程合理性。需收敛的是：本文的 WAPE 和变点检测在仿真中验证，尚不能说明真实飞行传感器噪声和未建模动态下同样稳定。

## 5. 贡献层

作者声明的贡献包括：在 ET-NMPG 中嵌入 WAPE 在线更新模型参数；针对再入过程保证路径约束；设计变点检测避免飞行中故障数据污染；通过数值仿真验证。

真实贡献有三层。第一是框架贡献，把事件触发 TOP、参数估计触发条件、最小估计数据量、最小估计间隔和模型更新串成完整在线制导流程。第二是估计方法贡献，对大气参数和气动/质量参数分别建立估计形式，并用权重自适应避免早期数据不足或参数敏感性差导致估计发散。第三是鲁棒性补强贡献，通过变点检测使参数突变后只使用合适时段数据，避免新旧参数混合污染估计。

创新强度属于工程算法整合型中等偏强。事件触发 MPC、NMPG、最小二乘参数估计都不是新概念；新意在于面向再入制导把它们组合到一个可运行框架，并用 Monte Carlo 证明成功率和终端误差改善。贡献边界是：仍依赖高质量状态/加速度/大气测量；参数模型形式预先给定；TOP 求解失败仍可能发生；阈值和权重参数需要调参。

## 6. 方法层

方法流程可以概括为：

```text
建立标准再入运动方程和带偏差/扰动真实模型
-> 离线获得初始最优参考轨迹和指导律
-> 飞行中比较实际状态与参考状态，计算事件触发条件
-> 触发时调用 NMPG 重解 TOP，更新指导律和参考轨迹
-> 若数据量与间隔满足条件，执行 WAPE 更新大气/气动/质量参数
-> 若检测到飞行中参数突变，重置或分段使用数据避免污染
-> 用更新模型继续事件触发制导
```

ET-NMPG 的事件条件是状态偏差超过阈值；NMPG 求解的是从当前状态到终端约束的在线轨迹优化，考虑路径约束和终端约束。WAPE 分为大气参数估计和气动参数估计。大气部分利用高度-密度关系做加权最小二乘；气动部分利用加速度观测和升阻模型估计 CL/CD 系数与质量相关参数。权重随可用数据长度、飞行阶段和参数敏感性调整，避免早期或弱激励数据主导估计。

方法匹配问题的关键是：事件触发解决“何时重算”，WAPE 解决“用什么模型重算”，变点检测解决“故障后哪些数据还能用”。关键参数包括状态误差阈值、最小估计数据量、最小估计时间间隔、WAPE 权重和路径约束松弛边界。复现信息较多，Table 1-3 给出算法参数、初末状态和模型参数偏差；但公式 OCR 噪声较重，真正复现需 PDF 原式和代码细节。

## 7. 证据层

证据链分三个层次。第一是典型案例对比：Fig. 5-6 显示无偏差时 FLG 和 ET-NMPG 都能成功，有大初始偏差时 FLG 失败而 NMPG 类方法可达终端；Fig. 7 显示 FC-NMPG 固定周期可能更新不及时导致路径约束失败，ET-NMPG 更能防止越界；Fig. 8-11 显示参数偏差下 ET-NMPG 可能多次 TOP 求解失败，而 WAPE-assisted ET-NMPG 通过参数更新保持在放宽约束内并延长触发间隔。

第二是 Monte Carlo 证据：300 个拉丁超立方样本中，ET-NMPG 成功 134 次，WAPE-assisted ET-NMPG 成功 213 次；Table 4 显示 WAPE-assisted ET-NMPG 的平均制导次数约 13.2，远少于 ET-NMPG 的 121.7，尽管单次求解时间更长，总仿真时间更短；Table 5 显示终端状态误差整体更小，尤其终端速度误差显著改善。Fig. 13-14 表明主要气动参数 CL0/CL1/CD0/CD1 估计较快收敛，二次/高阶项更难早期准确估计。

第三是飞行中故障证据：32 个突变样本中，变点检测能在不超过 3 s 误差内识别突变；Fig. 15-17 和 Table 6 显示不同突变时间下仍能满足轨迹和终端要求，早期故障更易通过 WAPE 平滑修正。

证据风险是：成功率和时间统计都来自 MATLAB 桌面仿真；传感器噪声、执行机构限制和真实气动不确定性简化；WAPE-assisted 方法单次 TOP 变慢，实时硬件可行性需要进一步证明。

论文结构是“传统制导困境 -> 事件触发框架 -> 参数估计补强 -> 完整框架 -> 仿真比较”。Introduction 用再入制导策略演化制造问题；Problem formulation 定义运动方程、偏差、扰动和制导问题；ET-NMPG section 讲 TOP 和在线求解技巧；WAPE section 讲参数估计和变点检测；Guidance framework 把 UI/FC/ET/WAPE-assisted 方法放在同一图里比较；Simulation section 先单例、再 Monte Carlo、再故障。

段落功能推进有工程算法论文的典型层次：先说明为何传统跟踪不够，再说明在线优化为何昂贵，再说明事件触发为何仍不够，最后引出参数估计。结果段也不是只报成功率，而是依次展示路径约束、地面轨迹、指导令、参数估计、终端误差和故障恢复。

语言画像偏控制工程和计算制导。高频词包括 `NMPG`、`event-triggered`、`trajectory optimization`、`parameter estimation`、`path constraints`、`guidance law`、`model deviation`、`in-flight faults`。典型句式是 “To address this issue, we propose...”、“It can be observed that...”、“the introduction of WAPE significantly...”。作者经常用 `balance`、`reduce`、`ensure`、`enhance` 来连接效率、精度和安全三类目标。

风险包括：第一，references.txt 编号不稳定，需 PDF 复核；第二，UI-NMPG 因仿真时间不现实没有实际比较，若结论说优于 UI-NMPG，应解释依据是计算资源而非同等实验结果；第三，WAPE 依赖可观测数据和预设参数模型，真实传感器误差可能影响估计；第四，事件阈值和权重参数由经验给定，缺少系统选择理论；第五，路径约束“放宽边界”需要解释工程可接受性；第六，仿真环境是 MATLAB 桌面，实时嵌入式可行性仍待验证。

审稿人可能追问：参数估计误差是否会引入不稳定指导令？变点检测在噪声下误报/漏报如何？鲁棒 NMPG 是否更合适？作者在结论中承认需进一步研究阈值选择和鲁棒 NMPG，这是合理收束。

## 10. 复用层

可复用选题角度：当一个在线优化算法存在“算得准但太频繁”问题时，可以先用事件触发减少调用，再用在线模型校正减少事件源头，形成“触发机制 + 模型自适应”的双层改进。

可复用 gap 表达：Event-triggered online optimization can reduce unnecessary replanning, but under model-parameter deviations the trigger frequency may increase and the optimization problem may become infeasible; online parameter estimation is therefore needed to update the model used by subsequent replanning.

可复用论证链：

```text
传统跟踪精度不足 -> 在线优化制导有潜力但成本高
-> 固定周期/不停更新各有问题 -> 事件触发减少不必要求解
-> 模型偏差导致频繁触发和求解失败
-> WAPE 在线修正模型 -> 变点检测处理突变
-> 单例/Monte Carlo/故障三层验证
```

## 2. 结构校验

所有逐篇报告均只包含以下十个 `##` 一级标题，且顺序一致：

```text
## 1. 身份层
## 2. 主张层
## 3. 选题层
## 4. 位置层
## 5. 贡献层
## 6. 方法层
## 7. 证据层
## 8. 结构语言层
## 9. 引用风险层
## 10. 复用层
```

摘要和结论已归入主张层；图表、公式和 Claim-Evidence 已归入证据层；逐段理解和语言画像已归入结构语言层；引用策略和审稿风险已归入引用风险层；可模仿/不可模仿内容已归入复用层。

## 3. 内容校验

每篇报告均覆盖以下信息类型：

- 论文身份、研究对象、方法组合、材料边界。
- 核心主张、摘要意译、结论意译、结论闭合度。
- 选题价值、问题来源、研究边界。
- 文献版图、gap 类型、本文位置。
- 作者声明贡献与真实贡献判断。
- 方法输入、输出、关键步骤、假设、复现边界。
- Claim-Evidence、关键图表、关键公式、实验/案例、证据强度。
- 章节推进、段落组功能、常用词句、语态、时态、语言风格。
- 引用策略、审稿风险、抽取风险。
- 可复用写法、不可直接模仿内容、迁移建议。

## 4. 已知抽取风险

本批报告基于 `801/cleantext` 抽取结果完成。以下问题在多篇论文中出现，已在对应报告中标注：

| 风险类型 | 影响 |
| --- | --- |
| metadata 作者缺失 | 多篇论文无法从 metadata 自动取得作者/机构，需要 PDF 首页核查 |
| 图题重复或缺失 | 图像细节、图号顺序和部分图表功能需 PDF 视觉核查 |
| 表格行列错位 | 表格中的具体数值、单位和列名需谨慎引用 |
| 表题过简 | 有些 `table_captions.txt` 只保留 Table 编号，需要从正文恢复功能 |
| 公式 OCR 噪声 | 复杂公式、下标、上下标和矩阵表达需 PDF 原式核对 |
| references 混入正文/结论/致谢 | 引文功能分析可用，但正式引用统计需回 PDF |
| abstract 混入正文 | 个别论文摘要抽取污染，报告中以 clean_body 交叉校正 |

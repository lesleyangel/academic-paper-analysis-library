# Batch 9 深度拆解总结

## 本批论文列表

1. `Weight-adaptive-parameter-estimation-assisted-event-t_2025_Aerospace-Science`

## 共同主题和方法谱系

本批只有 1 篇论文，主题集中在再入飞行器的在线模型预测制导。方法谱系是：再入运动建模 -> 在线轨迹优化 TOP -> 非线性模型预测制导 NMPG -> 事件触发 ET-NMPG -> 权重自适应参数估计 WAPE -> 突变点检测 -> Monte Carlo/故障仿真验证。

核心问题不是“如何离线求一条最优轨迹”，而是“真实再入过程中模型参数偏离参考值时，如何让在线优化制导仍然少调用、可行、准确并满足路径约束”。

## 常见 gap 类型

1. 工程部署 gap：传统轨迹跟踪制导在大初始偏差和欠驱动再入系统中容易产生状态散布。
2. 计算资源 gap：UI-NMPG 频繁优化，计算负担过大；FC-NMPG 固定周期更新，约束附近响应不及时。
3. 模型偏差 gap：ET-NMPG 虽能减少调用，但在气动/大气/质量参数偏差下会触发过密，甚至使 TOP 不可行。
4. 数据质量 gap：直接最小二乘在少数据、噪声和速度/攻角覆盖不足时会产生不合理气动参数。
5. 故障数据 gap：飞行中参数突变会污染历史样本，需要突变点检测后丢弃旧数据。

## 常见论证链

- 工程背景 -> 传统制导不足 -> NMPG 引入 -> UI/FC 策略局限 -> 事件触发缓解计算问题 -> 模型偏差造成新的失败模式 -> WAPE 在线修正模型 -> 突变检测防止数据污染 -> 典型算例 + Monte Carlo + 故障仿真验证。
- 方法模块 -> 对应失败模式：事件触发对应计算资源，WAPE 对应模型偏差，权重设计对应少数据和弱可辨识，change point detection 对应飞行中故障。
- 证据链：曲线展示机制，表格承载强 claim，统计样本支撑普遍性。

## 高频写作套路

- 用 “trade-off” 把精度和计算效率放在同一问题框架中。
- 用 “However” 连续制造二阶 gap：NMPG 有用但耗时，ET-NMPG 有用但受模型偏差影响，LS 有用但少数据下不稳。
- 用 “To address this” 把每个 gap 直接接到一个方法模块。
- 用 Remark 主动解释边界条件，例如数据不足不做 WAPE、参数更新过快会造成命令不平滑。
- 用 Table 4/5 这种指标表同时报告成功率、制导次数、计算时间和终端误差，避免单指标叙事。

## 可迁移到自己论文写作的资产

- 如果已有方法很成熟，可以从它在真实部署条件下的失败模式切入，而不是硬造“无人研究”的空白。
- 框架型创新最好让每个模块都有明确敌人：触发机制对抗无谓计算，估计机制对抗模型偏差，检测机制对抗数据污染。
- 参数估计或代理模型不要只证明自身误差低，要证明它改善了下游任务指标。
- 仿真验证可以分成三层：典型样例解释机制，Monte Carlo 给统计可靠性，故障/扰动样例证明鲁棒性。
- 主动承认部分参数难估、部分约束被放松、阈值仍需进一步研究，能让工程论文更可信。

## 本批总体观察

本批最有价值的写法是“二阶 gap 写法”：不是说已有方法没用，而是说明已有方法在真实模型偏差、路径约束和在线计算压力下会暴露新的失败模式。论文真正的贡献也不是 WAPE 估计本身，而是把模型校正纳入事件触发 NMPG，让制导调用频率、成功率、路径安全和终端精度一起改善。

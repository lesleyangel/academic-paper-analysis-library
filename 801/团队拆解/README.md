# 801 团队拆解总览

本目录按照 `个人团队拆解分析模板.md`，基于 `801/深度拆解` 的 49 篇逐篇论文拆解和 9 个 batch summary，对 801 样本背后的作者/团队型论文生产系统做横向分析。

## 文件说明

- `801团队拆解分析报告.md`：正式总报告，覆盖样本范围、基本身份、研究轨迹、方向矩阵、贡献栈、技术画像、证据风格、写作画像、合作网络、期刊策略、发表公式、团队类型和可学习资产。
- `801逐篇团队视角索引.md`：49 篇论文逐篇索引，说明每篇在团队轨迹中的位置、方向标签和新增/复用资产。
- `_team_analysis_instructions.md`：本轮 multiagent 团队拆解任务说明。
- `agent_notes/`：5 个 agent 的独立维度笔记，分别覆盖研究轨迹、贡献/技术/证据、写作画像、合作/期刊策略、发表公式/可学习资产。

## 核心结论

801 样本最像一个“高超声速与航天复杂系统的中间表示生产线”：它长期把热、气动、结构、材料、控制和不确定性中的高成本/不可观测/强耦合问题，压缩成单胞、等效层、POD/ROM、代理模型、稀疏传感映射或阶段化优化流程，再用工程对象和定量指标证明这些中间层能进入设计、评估或在线决策。

## 证据边界

- 研究方向、期刊策略、方法体系、写作风格：证据较强。
- 核心作者与主机构：可判断高频作者和主机构线索，但不能据此推断唯一 PI 或内部组织结构。
- 合作功能、产业数据、学生梯队、特殊期刊关系：证据不足时均标注为不可判断或推断。

## 建议读法

1. 先读 `801团队拆解分析报告.md` 的 `1-5` 节，建立团队整体画像和方向地图。
2. 再读 `7-13` 节，理解它为什么能持续发表。
3. 如果想学习写作，重点看 `10. 写作画像` 和 `15. 可学习资产`。
4. 如果想追踪具体论文，打开 `801逐篇团队视角索引.md`，按资产链条阅读对应单篇深拆。

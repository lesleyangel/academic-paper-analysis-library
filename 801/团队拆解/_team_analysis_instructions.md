# 801 团队拆解任务说明

本目录用于按照 `个人团队拆解分析模板.md`，基于 `801/深度拆解`、`801/文本/_manifest.json` 和必要的 `801/文本/txt` 抽取文本，对 801 论文集合背后的团队/作者群体做“学术生产系统”拆解。

## 输入资料

- 模板：`个人团队拆解分析模板.md`
- 单篇深拆：`801/深度拆解/papers/*.md`
- 批次总结：`801/深度拆解/batch_summaries/*.md`
- 论文元数据：`801/文本/_manifest.json`
- 必要时查原始抽取文本：`801/文本/txt/*.txt`

## 输出位置

- agent 分析笔记：`801/团队拆解/agent_notes/*.md`
- 总报告：`801/团队拆解/801团队拆解分析报告.md`
- 逐篇作者视角索引：`801/团队拆解/801逐篇团队视角索引.md`
- 总览：`801/团队拆解/README.md`

## 分析边界

- 只基于论文、深拆和元数据做证据化判断。
- 不做作者性格猜测，不臆断未公开的资源或关系。
- 合作功能、核心 PI、团队边界如证据不足，要标注“推断”或“不可判断”。
- 原始 PDF 和全文抽取文本不进入公开报告；报告只写分析、归纳和可复用资产。

## 总报告必须覆盖

1. 一句话画像
2. 样本范围
3. 基本身份
4. 研究轨迹
5. 方向矩阵
6. 选题风格
7. 贡献栈
8. 技术与理论画像
9. 证据风格
10. 写作画像
11. 合作网络
12. 期刊策略
13. 为什么能发表
14. 类型判断
15. 可学习资产
16. 对我自己的启发

## 证据强度标注

- 强证据：多篇论文反复出现，且跨批次/跨年份稳定。
- 中证据：某一阶段或某一方向反复出现。
- 弱证据：单篇或少数论文出现，可能来自合作方。
- 不可判断：现有论文不足以支持。

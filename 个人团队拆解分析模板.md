# 个人/团队拆解分析模板

这份文档用于以个人或研究团队为中心拆解论文。和单篇论文拆解不同，这里的核心问题不是“这篇论文如何成立”，而是“这个人或团队如何持续生产论文”：他们长期研究什么、如何定义问题、如何积累方法和数据、如何选择期刊、如何组织合作、如何形成稳定写作风格，以及为什么他们能持续发表。

基本单位仍然是每篇论文。每篇论文都是作者/团队行为的一次外显记录。

## 一、核心原则

拆个人/团队要看三件事：

1. **研究轨迹**：他们的研究方向如何演化，哪些问题一直没变，哪些方法在升级。
2. **生产系统**：他们依靠什么资源持续出成果：理论、设备、数据、代码、案例、合作网络、学生梯队。
3. **表达系统**：他们如何把自己的积累包装成期刊能接受的标题、结构、贡献和证据。

不要把作者画像写成性格猜测。只分析公开论文能支持的学术行为：

- 他/她/他们反复选择什么问题？
- 反复使用什么理论、方法、对象、数据和场景？
- 论文之间是否存在连续升级关系？
- 哪些贡献是团队长期资产，而不是单篇偶然发挥？
- 他们在合作网络里扮演什么角色？
- 他们为什么能在某些期刊上发表？
- 我能学习他们的什么，不能模仿什么？

### 2. 样本量建议

```text
个人快速画像：5-8 篇
个人标准画像：15-25 篇
团队标准画像：20-50 篇
深度谱系分析：50 篇以上
```

### 3. 证据等级

| 等级 | 含义 |
| --- | --- |
| 强证据 | 多篇论文反复出现，且作者角色稳定 |
| 中证据 | 某一阶段或某一方向反复出现 |
| 弱证据 | 单篇出现，可能是合作方贡献 |
| 不可判断 | 论文不足以支持推断 |

| 与后文关系 | 是否成为后续论文基础 |
| 目标期刊匹配 | 为什么投/能发这个期刊 |

## 五、研究轨迹拆解

### 1. 时间线

```text
阶段 1：年份范围 / 关键词 / 代表论文 / 主要贡献
阶段 2：年份范围 / 关键词 / 代表论文 / 主要贡献
阶段 3：年份范围 / 关键词 / 代表论文 / 主要贡献
```

观察问题：

- 研究方向是线性深化，还是多线并行？
- 是否存在明显转向？
- 转向由什么驱动：新方法、新数据、新合作、新期刊、新工程需求？
- 哪些问题贯穿多个阶段？

### 2. 稳定母题

每个成熟作者/团队通常有一个或几个母题。

```text
长期母题 1：
长期母题 2：
长期母题 3：
母题背后的核心矛盾：
母题为什么能持续产出论文：
```

例子：

- 一直围绕“预测精度”升级方法。
- 一直围绕“机制解释”升级模型。
- 一直围绕“工程系统信息表示”扩展场景。
- 一直围绕“多尺度桥接”连接实验和理论。

### 3. 方向矩阵

| 方向 | 代表论文 | 对象 | 方法 | 贡献 | 期刊 | 阶段 |
| --- | --- | --- | --- | --- | --- | --- |
|  |  |  |  |  |  |  |

判断：

```text
主方向：
副方向：
试探方向：

最可能继续发展的方向：
```

```text
1. 这个人/团队一句话擅长什么？
2. 他们长期研究的母题是什么？
3. 他们的方向如何演化？
4. 他们最常用什么理论和方法？
5. 他们依靠什么独特资源？
6. 他们最常制造什么 gap？
7. 他们最常提供什么贡献？
8. 他们的证据风格是什么？
9. 他们的标题、摘要和 Introduction 有什么稳定写法？
10. 他们通常投什么期刊，为什么匹配？
11. 他们为什么能持续发表？
12. 我能学什么，不能学什么？
```

## 十九、一句话总结

拆个人/团队，不是给作者贴标签，而是把论文当作连续证据，复原一个学术生产系统：他们如何长期选择问题，如何积累可复用资产，如何组织合作，如何把资源转化成目标期刊认可的论文，以及我们能从这套系统里迁移什么。

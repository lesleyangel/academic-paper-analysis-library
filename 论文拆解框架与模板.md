# 论文拆解框架与模板

这份文档用于系统拆解一篇学术论文。它的目标不是把论文“总结一遍”，而是还原一篇论文如何完成学术说服：它为什么值得写、凭什么算新、如何组织证据、如何布局篇章、如何使用文笔和引用让读者相信它。

适用场景：

- 精读高水平期刊论文。
- 学习某个期刊的选题、写法、贡献表达和引用习惯。
- 为自己的论文选题、Introduction、Related Work、Discussion 提供模仿对象。
- 建立个人或团队的“论文写作素材库”。
- 训练审稿视角，识别一篇论文的强项、漏洞和可复用套路。

## 一、核心原则

拆论文要同时看三件事：

1. **内容系统**：它研究什么问题，提出什么方法，得到什么结论。
2. **说服系统**：它如何证明这个问题重要、这个方法必要、这个结论可信。
3. **写作系统**：它如何用结构、文笔、图表、引用和语气把贡献包装成一篇成熟论文。

不要只问“这篇论文讲了什么”，而要追问：

- 作者为什么这样定义问题？
- 作者如何制造研究缺口？
- 作者如何把自己的工作放到已有研究之后？
- 作者的创新究竟是哪一类？
- 证据是否刚好支撑主张？
- 哪些句式、段落、图表、引用方式值得复用？
- 如果我是审稿人，我最可能质疑哪里？

## 二、拆解层级总览

建议按 12 个层级拆解：

1. **论文身份层**：期刊、领域、类型、对象、问题、方法、材料、数据。
2. **选题层**：研究问题从哪里来，为什么值得研究。
3. **领域位置层**：它和已有研究是什么关系。
4. **创新层**：它到底新在哪里，新意是否必要。
5. **论证层**：问题、方法、证据、结论之间是否闭合。
6. **方法层**：理论、模型、实验、算法或案例如何支撑主张。
7. **证据层**：数据、图表、公式、对比、消融、统计是否足够。
8. **结构层**：各章节如何分工，篇章节奏如何推进。
9. **文笔层**：常用词、句式、语气、段落推进和表达习惯。
10. **引用层**：引用密度、引用姿态、文献组织和 gap 制造方式。
11. **审稿风险层**：最容易被质疑的地方是什么。
12. **可复用资产层**：哪些选题角度、论证套路、句式和图表表达能迁移到自己的写作中。

## 三、三种拆解深度

### 1. 快速拆解：15-30 分钟

适合判断一篇论文值不值得精读。

需要回答：

- 论文一句话主张是什么？
- 研究对象是什么？
- 核心问题是什么？
- 主要方法是什么？
- 最大创新是什么？
- 最关键图表是哪几个？
- 这篇论文值得学什么？

输出建议：

```text
一句话主张：
研究问题：
核心创新：
关键证据：
最值得学习的地方：
可能的缺陷：
```

### 2. 标准拆解：1-2 小时

适合做文献笔记、组会汇报、论文复盘。

需要拆解：

- Introduction 如何制造问题。
- Related Work 如何划分领域。
- Method 如何支撑创新。
- Results 如何验证主张。
- Discussion 如何扩大意义并控制边界。
- 作者常用表达和引用方式。

输出建议：

```text
论文定位：
问题链条：
创新分类：
章节结构：
证据链：
引用策略：
文笔画像：
可复用模板：
审稿风险：
```

### 3. 精细拆解：半天到一天

适合学习顶刊写作、模仿目标期刊、训练论文生产能力。

需要逐段、逐图、逐句拆：

- 每一段承担什么功能。
- 每一句如何推进论证。
- 每个引用放在什么位置，承担什么作用。
- 每个图表对应哪个 claim。
- 每个贡献是否被方法和证据支撑。
- 每个章节结尾如何引出下一部分。
- 论文的语气强弱如何控制。

输出建议：

```text
段落功能表：
Claim-Evidence 对照表：
图表功能表：
引用功能表：
常用句型库：
术语与概念表：
审稿人质疑清单：
可模仿写法：
不可模仿问题：
```

## 四、论文身份层

先建立论文的基本画像。

需要记录：

- 标题。
- 作者与机构。
- 期刊或会议。
- 发表年份。
- 学科领域。
- 论文类型：理论、方法、实验、数值模拟、综述、案例、数据集、应用、观点。
- 研究对象：材料、结构、系统、算法、现象、机制、群体、文本、样本。
- 主要方法：实验、建模、仿真、推导、调查、案例比较、机器学习、统计分析。
- 主要证据：数据、图表、公式、案例、实验结果、对比结果、文献综合。
- 目标读者：领域专家、方法研究者、工程应用者、跨学科读者。

关键问题：

- 这篇论文属于“提出新问题”“提出新方法”“解释新机制”“验证旧理论”“整合已有研究”中的哪一种？
- 它更像是领域贡献、方法贡献、理论贡献，还是工程贡献？
- 它面向的是学术前沿，还是应用需求？

## 五、选题层

选题层关注：这篇论文为什么值得写。

### 1. 问题来源

判断问题从哪里来：

- 来自工程瓶颈。
- 来自理论空白。
- 来自实验现象无法解释。
- 来自已有模型不适用。
- 来自新材料、新场景、新数据。
- 来自测量、预测、控制、优化、解释等需求。
- 来自跨学科迁移。

可填写：

```text
问题来源：
现实背景：
学术背景：
已有方法的不足：
本文切入点：
```

### 2. 问题定义

要看作者如何把一个大问题收束成可研究问题。

重点观察：

- 是否从大背景收束到具体对象。
- 是否给出明确研究边界。
- 是否把问题表达成可验证、可建模、可比较、可解释的形式。
- 是否避免过大、过泛、过空的命题。

常见问题表达：

```text
X remains poorly understood because...
The mechanism underlying X is still unclear.
Existing models fail to capture...
Despite extensive studies on X, Y remains underexplored.
There is a need for a quantitative framework to...
```

### 3. 选题价值

判断它的价值来自哪里：

- 理论价值：解释机制、统一现象、扩展模型。
- 方法价值：提高预测、测量、设计、控制能力。
- 实证价值：提供新数据、新观测、新对比。
- 工程价值：解决实际场景中的性能、可靠性、效率问题。
- 学科价值：连接两个原本分离的研究方向。

拆解问题：

- 这个问题如果解决，会改变什么？
- 这个问题为什么以前没有被充分解决？
- 作者是否说明了“现在做这件事”的必要性？
- 问题的重要性是靠事实、引用、案例还是直觉支撑？

## 六、领域位置层

这一层看论文如何把自己放进已有研究版图。

### 1. 文献版图

拆 Related Work 或 Introduction 中的文献组织方式。

常见组织方式：

- 按时间线组织：早期研究、近期进展、最新瓶颈。
- 按方法流派组织：理论模型、实验方法、数值模拟、数据驱动方法。
- 按问题类别组织：预测、解释、控制、优化、表征。
- 按尺度组织：微观、介观、宏观、多尺度。
- 按材料或对象组织：不同材料体系、结构类型、应用场景。
- 按优缺点组织：已有方法分别能解决什么、不能解决什么。

可填写：

```text
作者把已有研究分成了几类：
每一类代表性文献：
每一类解决了什么：
每一类留下了什么不足：
本文位于哪一类之后：
本文试图跨越哪几类：
```

### 2. 学术站位

判断作者如何处理与前人关系：

- 继承：建立在某个经典理论或方法之上。
- 修正：指出已有理论在某些边界下不成立。
- 扩展：把方法推广到新对象、新尺度、新场景。
- 对比：证明本文方法优于某类方法。
- 整合：把分散结论统一到一个框架。
- 替代：提出新的解释路径。

关键句型：

```text
Building on previous work on X, we...
Unlike existing approaches that..., our method...
While prior studies have focused on X, less attention has been paid to Y.
These studies provide important insights, but they do not address...
```

### 3. Gap 制造方式

重点看作者如何证明“还缺一块”。

常见 gap 类型：

- 对象 gap：某类材料、系统、群体、条件未被研究。
- 机制 gap：现象被观察到，但原因不清楚。
- 方法 gap：已有方法不能测量、预测、解释或控制目标问题。
- 尺度 gap：微观机制和宏观表现之间缺桥梁。
- 数据 gap：缺少足够数据或高质量数据。
- 场景 gap：已有研究只在理想条件下成立。
- 理论 gap：已有理论无法统一多个现象。
- 验证 gap：方法提出了，但缺少系统验证。

可填写：

```text
作者声称的 gap：
gap 属于哪一类：
gap 是否真实：
gap 是否足够重要：
gap 是否被文献支撑：
如果我是审稿人，会不会接受这个 gap：
```

## 七、创新层

创新层要回答：这篇论文凭什么算新。

### 1. 创新类型

可从以下类别判断：

- 问题创新：提出或重新定义一个值得研究的问题。
- 理论创新：提出新的理论解释、概念框架或机制模型。
- 方法创新：提出新的实验方法、算法、模型、分析流程。
- 数据创新：构建新数据集，获得新观测，覆盖新条件。
- 证据创新：用更强证据验证、反驳或修正已有观点。
- 应用创新：把已有方法应用到新的高价值场景。
- 整合创新：把分散研究组织成统一框架。
- 表达创新：用新的指标、图式、公式或可视化方式重述问题。

可填写：

```text
作者声明的创新：
我判断的真实创新：
创新类型：
创新强度：强 / 中 / 弱
创新是否必要：
创新是否被充分验证：
创新是否容易被前人工作挑战：
```

### 2. 贡献表达

看作者如何写 contribution。

常见贡献句式：

```text
The main contributions of this work are threefold.
First, we develop...
Second, we demonstrate...
Third, we reveal...
This work provides a quantitative framework for...
Our results establish a connection between X and Y.
The proposed approach enables...
```

拆解问题：

- 贡献是按方法、发现、应用来写，还是按章节顺序写？
- 贡献是否可验证？
- 贡献是否过度夸大？
- 每个贡献后文是否都有对应证据？
- 摘要、Introduction、Conclusion 中的贡献是否一致？

### 3. 创新可信度

判断创新是否站得住：

- 是否和最近文献比较过。
- 是否说明与经典方法的差异。
- 是否有定量提升或机制解释。
- 是否有充分实验、推导或案例支撑。
- 是否说明适用边界。
- 是否承认局限。

审稿人会问：

- 这真的是新的，还是换了说法？
- 和最接近的前人工作相比，新在哪里？
- 新方法是否比旧方法更好，还是只是更复杂？
- 新发现是否有足够证据？
- 贡献是否只是特定数据或特定参数下成立？

## 八、论证层

论证层关注整篇论文的逻辑闭环。

### 1. 问题链条

一篇成熟论文通常有这样的链条：

```text
领域背景
→ 重要现象或工程需求
→ 已有研究进展
→ 未解决 gap
→ 本文研究问题
→ 本文方法
→ 本文证据
→ 本文结论
→ 理论或实践意义
```

拆解时逐项检查：

```text
背景是否必要：
gap 是否从背景自然推出：
研究问题是否具体：
方法是否针对 gap：
证据是否验证方法：
结果是否支持结论：
结论是否回到开头问题：
意义是否没有超出证据：
```

### 2. Claim-Evidence 对照

把重要主张和证据对应起来。

模板：

| Claim | 位置 | 支撑证据 | 证据类型 | 强度 | 风险 |
| --- | --- | --- | --- | --- | --- |
| 论文声称什么 | 摘要/引言/讨论 | 图、表、公式、实验、引用 | 数据/推导/比较/案例 | 强/中/弱 | 过度推广/证据不足/缺引用 |

重点检查：

- 摘要里的每个强 claim 是否在正文中被证明。
- Introduction 的 gap 是否被 Related Work 支撑。
- Results 的结果是否能推出 Discussion 的机制解释。
- Conclusion 是否引入了正文没有证明的新 claim。

### 3. 因果与机制

如果论文解释机制，要特别检查：

- 作者区分了相关性和因果性吗？
- 是否有控制实验、对照组、消融分析或理论推导？
- 机制解释是否只是猜测？
- 图表是否能支撑机制路径？
- 是否讨论替代解释？

常见机制表达：

```text
These results suggest that...
This behavior can be attributed to...
The observed trend is consistent with...
One possible explanation is...
This indicates a transition from X to Y.
```

## 九、方法层

方法层要判断：作者的方法是否真的能回答问题。

### 1. 方法类型

先判断方法属于哪一类：

- 理论推导。
- 数值模拟。
- 物理实验。
- 数据驱动模型。
- 统计分析。
- 案例研究。
- 文献综述与框架构建。
- 多方法组合。

可填写：

```text
方法名称：
方法目标：
输入：
输出：
关键假设：
关键参数：
适用范围：
不可适用范围：
```

### 2. 方法必要性

问：

- 为什么必须用这个方法？
- 旧方法为什么不够？
- 方法复杂度是否和问题难度匹配？
- 方法是否为了创新而创新？
- 方法是否能被复现？

### 3. 方法结构

对于模型、算法、理论框架，拆：

- 总体框架。
- 模块划分。
- 变量定义。
- 假设条件。
- 控制方程或核心公式。
- 参数识别。
- 训练或求解流程。
- 验证流程。
- 敏感性分析。
- 边界条件。

对于实验论文，拆：

- 样品或对象。
- 实验装置。
- 测量指标。
- 工况设置。
- 对照设计。
- 重复次数。
- 误差来源。
- 数据处理方式。
- 结果可靠性。

对于综述论文，拆：

- 文献筛选标准。
- 分类维度。
- 综述边界。
- 组织逻辑。
- 归纳框架。
- 未来问题。

## 十、证据层

证据层看：作者如何让读者相信。

### 1. 证据类型

常见证据：

- 实验数据。
- 理论推导。
- 数值模拟。
- 统计检验。
- 消融实验。
- 对比实验。
- 参数敏感性分析。
- 案例验证。
- 可视化图像。
- 文献支持。
- 误差分析。
- 失败案例或边界案例。

### 2. 图表拆解

每个关键图表都要问：

- 这张图想证明什么？
- 它对应哪个 claim？
- 横轴、纵轴、颜色、标注各自代表什么？
- 图中最重要的趋势是什么？
- 是否有对照组？
- 是否有误差条或统计显著性？
- 是否有替代解释？
- 图注是否足够独立理解？
- 图表顺序是否符合论证节奏？

图表功能模板：

| 图表 | 功能 | 对应 claim | 关键发现 | 支撑强度 | 可学习之处 |
| --- | --- | --- | --- | --- | --- |
| Figure 1 | 问题设定/方法框架/结果证明 |  |  | 强/中/弱 |  |

### 3. 公式拆解

对于含公式论文，拆：

- 每个符号是否定义清楚。
- 公式来自推导、引用还是经验拟合。
- 公式解决什么问题。
- 公式和物理意义、统计意义或算法意义是否对应。
- 公式中的假设是否明确。
- 后文是否使用这个公式产生结果。

公式模板：

```text
公式编号：
核心含义：
变量定义：
前提假设：
推导来源：
服务的 claim：
可能的边界条件：
```

### 4. 证据强度

可以按四级判断：

- 强：有多组独立证据，且排除了主要替代解释。
- 中：有直接证据，但范围有限。
- 弱：只有间接证据、单个案例或描述性结果。
- 不足：claim 大于证据，或证据与 claim 不匹配。

## 十一、结构层

结构层看篇章如何组织。

### 1. 摘要

拆摘要时看五件事：

- 背景：为什么这个问题重要。
- 缺口：已有研究缺什么。
- 方法：本文做了什么。
- 结果：发现了什么。
- 意义：这些发现意味着什么。

模板：

```text
摘要第一句功能：
问题句：
方法句：
结果句：
贡献句：
是否有过度 claim：
```

### 2. Introduction

常见结构：

```text
大背景
→ 具体问题
→ 已有进展
→ 未解决缺口
→ 本文目标
→ 本文方法
→ 本文贡献
→ 文章结构
```

重点拆：

- 第一段如何抓住领域重要性。
- 第二段如何收束到具体问题。
- 文献是如何被分类的。
- gap 句在哪里出现。
- 作者如何从 gap 过渡到本文方案。
- 贡献是否清晰列出。
- 结尾是否预告文章结构。

常见句式：

```text
X plays a critical role in...
Recent studies have shown that...
However, the relationship between X and Y remains unclear.
To address this gap, this study...
Here, we propose...
```

### 3. Related Work

拆：

- 是否按主题而不是逐篇罗列。
- 是否对每类研究都给出评价。
- 是否有明确的“已有研究不足”。
- 是否自然引出本文位置。
- 是否引用了最接近的工作。

常见段落结构：

```text
主题句：某类研究解决了什么。
文献组：列出代表性研究。
评价句：这些研究的价值。
不足句：它们还没有解决什么。
过渡句：因此本文需要处理什么。
```

### 4. Method / Theory / Framework

拆：

- 是否先给全局框架，再进细节。
- 是否说明假设和边界。
- 是否解释变量、参数、符号。
- 是否把方法步骤和研究问题对应。
- 是否有可复现信息。

### 5. Results

拆：

- 结果顺序是否从基础验证到核心发现。
- 每个小节是否回答一个研究问题。
- 是否先描述现象，再解释机制。
- 是否避免把讨论提前塞入结果。
- 是否用图表而不是文字堆砌。

常见结果句式：

```text
Figure X shows that...
As X increases, Y decreases...
Compared with..., the proposed method...
This trend becomes more pronounced when...
```

### 6. Discussion

拆：

- 是否解释结果背后的机制。
- 是否把结果放回领域语境。
- 是否讨论理论意义。
- 是否讨论实践意义。
- 是否承认局限。
- 是否提出未来研究。

注意区分：

- Observation：观察到了什么。
- Interpretation：这说明什么。
- Mechanism：为什么会这样。
- Implication：这对领域有什么意义。
- Limitation：还不能说明什么。

### 7. Conclusion

拆：

- 是否回到研究问题。
- 是否压缩主要发现。
- 是否避免引入新证据。
- 是否控制贡献边界。
- 是否给出未来工作或应用前景。

常见结论句式：

```text
In this study, we investigated...
The results show that...
These findings suggest that...
This work provides...
Future work should...
```

### 8. 章节结构与章节名分析

这一层不是只看论文有哪些章节，而是看作者如何用章节名组织读者预期。章节名本身是一种“路线图”：它告诉读者这篇论文是按问题推进、按方法模块推进、按实验结果推进，还是按机制发现推进。

重点拆：

- 章节数量：全文分成几大节，是否符合期刊常见结构。
- 章节顺序：是 IMRaD 顺序，还是理论/模型/验证/应用的顺序。
- 章节名类型：描述型、问题型、方法型、结果型、机制型、贡献型。
- 章节名粒度：是宽泛标题，还是把具体对象、变量、机制写出来。
- 小节层级：是否使用二级、三级标题；层级是否过深。
- 小节功能：每个小节是否回答一个明确问题。
- 标题并列性：同级标题的语法结构是否一致。
- 标题推进性：标题之间是否形成从问题到证据再到意义的递进。
- 标题信息量：标题是否只写“Results”，还是写出具体发现。
- 标题风格：偏工程流程、理论推导、实验验证，还是机制叙事。

章节名常见类型：

```text
描述型：Materials and methods / Numerical model / Experimental setup
问题型：Effect of X on Y / Role of X in Y / Influence of boundary conditions
结果型：X enhances Y / X delays the onset of Y / Scaling law for X
机制型：Mechanism of X / Transition from X to Y / Coupled evolution of X and Y
验证型：Model validation / Comparison with experiments / Sensitivity analysis
应用型：Application to X / Design implications / Practical guidelines
```

可填写：

```text
章节总数：
章节顺序：
是否标准 IMRaD：
章节名主要类型：
标题是否并列：
标题是否具体：
标题是否暴露核心变量：
标题是否提前透露结论：
最值得模仿的章节名：
最弱的章节名：
如果改写标题，可以如何改：
```

### 5. 标题、小标题和图注

标题拆：

- 是否包含对象。
- 是否包含方法。
- 是否包含机制。
- 是否包含结论。
- 是否过度承诺。

小标题拆：

- 是描述内容，还是直接表达观点？
- 是否形成清晰逻辑顺序？
- 是否帮助读者快速理解论文贡献？

### 6. 词频、词类与语法统计

词频分析不是为了机械数词，而是为了识别论文的“语言指纹”：作者反复使用哪些概念、动作、限定、评价和连接方式。它能帮助你判断论文的关注中心、论证习惯、语气强弱和期刊风格。

#### 词频统计

建议至少统计：

- 高频学术名词：problem, model, mechanism, framework, parameter, behavior, response, deformation, transition。
- 高频领域名词：材料、结构、算法、实验对象、物理量、尺度、边界条件。
- 高频动词：propose, develop, derive, show, demonstrate, reveal, suggest, indicate, validate, compare。
- 高频形容词：effective, significant, robust, nonlinear, local, global, critical, dominant, quantitative。
- 高频副词：significantly, substantially, systematically, experimentally, numerically, theoretically, approximately。
- 高频连接词：however, therefore, moreover, furthermore, whereas, consequently, in contrast。
- 高频限定词：may, might, can, could, likely, partially, generally, approximately。
- 高频否定或不足词：lack, limited, unclear, unresolved, insufficient, challenging。
- 高频 n-gram：two-word 或 three-word 短语，如 `phase-field model`, `mechanical response`, `finite element`, `stress field`。

可填写：

#### 主动、被动与句法习惯

重点看作者如何分配责任和动作：

- 主动语态：常用于本文贡献、方法动作、作者主动选择，例如 `we develop`, `we propose`, `we demonstrate`。
- 被动语态：常用于实验流程、样品制备、模型求解、客观结果，例如 `samples were prepared`, `the equations are solved`。
- 名词化：把动作压缩成学术名词，例如 `investigate` 变成 `investigation`，`validate` 变成 `validation`。
- 长句结构：是否经常用从句、插入语、并列短语解释条件。
- 短句结构：是否在结果和贡献处用短句增强清晰度。
- 句首结构：是否常用 `To address...`, `Based on...`, `Compared with...`, `As shown in...`。

```text

```

#### 时态分析

时态能暴露论文如何区分“领域共识、本文操作、本文发现和未来意义”。

常见规律：

- 一般现在时：用于领域事实、模型定义、图表说明、论文主张。
- 一般过去时：用于本文实验、仿真、数据处理和已完成结果。
- 现在完成时：用于总结已有研究进展，如 `has been studied`, `have shown`。
- 将来时或情态表达：用于未来工作、潜在应用和未验证推论。

可填写：

```text

```

#### 学术名词、形容词与副词

这部分看论文如何塑造“学术质感”和“claim 强度”。

学术名词可分：

- 对象名词：material, structure, system, specimen, interface。
- 方法名词：model, framework, algorithm, simulation, experiment。
- 机制名词：mechanism, transition, instability, localization, coupling。
- 证据名词：result, observation, comparison, validation, benchmark。
- 边界名词：assumption, limitation, condition, regime, range。

形容词可分：

- 强度形容词：significant, substantial, pronounced, critical。
- 性质形容词：nonlinear, anisotropic, heterogeneous, dynamic。
- 范围形容词：local, global, microscale, macroscale。
- 稳健形容词：robust, consistent, stable, reliable。

副词可分：

- 强化副词：significantly, substantially, notably。
- 谨慎副词：approximately, partially, potentially, likely。
- 方法副词：numerically, experimentally, theoretically, analytically。
- 递进副词：furthermore, additionally, consequently。

拆解问题：

- 高频名词是否与核心贡献一致？
- 形容词是在精确限定，还是在空泛拔高？
- 副词是在提供方法信息，还是在增强语气？
- `significantly` 是否有统计或量化支撑？
- `robust` 是否有多条件验证支撑？

#### 词频与章节的关系

不要只看全文词频，也要看词在各章节中的分布。

可拆：

- Abstract 高频词：是否对应论文卖点。
- Introduction 高频词：是否围绕背景、gap、challenge。
- Method 高频词：是否围绕模型、变量、参数、假设。
- Results 高频词：是否围绕趋势、比较、验证、性能。
- Discussion 高频词：是否围绕机制、意义、局限。
- Conclusion 高频词：是否回到核心贡献。

## 十三、引用层

引用层看作者如何借助文献建立可信度、制造 gap、控制学术位置。

### 1. 引用密度

观察：

- 哪些段落引用最多。
- 是一句一引，还是一段集中引用。
- 是否在强 claim 后及时引用。
- 是否用综述文献支撑大背景。
- 是否用最新文献支撑前沿性。
- 是否用经典文献支撑基本概念。

### 2. 引用功能

每个引用可以标注功能：

- 定义概念。
- 支撑背景。
- 证明重要性。
- 说明已有方法。
- 划分研究流派。
- 指出不足。
- 对比本文方法。
- 继承理论基础。
- 提供数据或参数来源。
- 支撑讨论中的解释。

模板：

| 引用 | 位置 | 功能 | 姿态 | 是否关键 | 备注 |
| --- | --- | --- | --- | --- | --- |
| Author, Year | Introduction | 背景/缺口/方法/对比 | 认同/补充/转折/批判 | 是/否 |  |

### 3. 引用姿态

判断作者对前人是什么态度：

- 认同：前人证明了某事实。
- 继承：本文建立在前人框架上。
- 补充：前人解决了部分问题。
- 转折：前人有价值，但还不够。
- 批判：前人方法存在局限。
- 对比：本文与前人方法不同。

常见引用句式：

```text
Previous studies have demonstrated that...
Several approaches have been developed to...
These methods provide valuable insights into...
However, they are limited by...
In contrast to prior work, this study...
```

### 4. 引用偏好

总结作者或期刊偏好：

- 是否偏好近五年文献。
- 是否保留大量经典理论文献。
- 是否喜欢引用综述。
- 是否引用同一期刊文章。
- 是否引用跨学科文献。
- 是否集中引用某几个研究团队。
- 是否强调方法论文还是应用论文。

可填写：

```text
近五年文献比例：
经典文献数量：
综述文献数量：
同刊引用数量：
最常引用的研究群体：
引用主要服务于：背景 / gap / 方法 / 对比 / 讨论
```

### 5. Gap 与引用的关系

一个好的 gap 通常不是凭空声明，而是通过引用构造出来。

拆解方式：

```text
引用组 A 说明：已有研究已经解决了什么。
引用组 B 说明：某类方法仍有不足。
引用组 C 说明：本文问题尚未被系统研究。
因此本文提出：
```

重点判断：

- gap 前是否有足够文献铺垫。
- 作者是否只引用对自己有利的文献。
- 是否遗漏最接近的反例。
- 是否把文献不足说得太绝对。

## 十四、审稿风险层

用审稿人视角找漏洞。

常见高风险问题：

- 标题或摘要承诺过大，正文证明不足。
- Introduction 背景很多，但 gap 不清楚。
- Related Work 只是罗列，没有形成领域版图。
- 创新点没有和最接近文献比较。
- 方法复杂，但必要性不足。
- 实验或数据只覆盖狭窄条件。
- 结果描述充分，但机制解释薄弱。
- Discussion 过度推广。
- Conclusion 提出正文没有证明的新主张。
- 关键术语前后不一致。
- 重要 claim 缺引用、缺实验或缺推导。

审稿人问题清单：

```text
这篇论文的核心贡献是否一眼可见？
gap 是否真实且重要？
最接近的前人工作是否被公平讨论？
方法是否必要且可复现？
证据是否足以支持最强 claim？
是否存在替代解释？
是否有边界条件和局限讨论？
图表是否支撑结论，而不是只展示结果？
引用是否覆盖关键文献？
论文是否有明显 scope drift？
```

## 十九、最小可用版本

如果时间很少，只保留这 12 个问题：

```text
1. 这篇论文的一句话主张是什么？
2. 它解决的核心问题是什么？
3. 它如何证明这个问题重要？
4. 它把已有研究分成了哪几类？
5. 它的 gap 是什么，是否真实？
6. 它的创新属于哪一类？
7. 它用什么证据支撑最重要的 claim？
8. 它的篇章结构如何推进说服？
9. 它的章节名如何组织读者预期？
10. 它的高频词、语态、时态和词类有什么稳定习惯？
11. 它的文笔和引用有什么稳定习惯？
12. 我能从中复用什么，不能复用什么？
```

## 二十、一句话总结

拆解一篇论文，不是把内容压缩成笔记，而是把它拆成一个完整的学术生产样本：选题如何成立，创新如何声明，证据如何支撑，结构如何推进，文笔如何包装，引用如何站位，以及哪些东西可以真正迁移到自己的论文写作中。

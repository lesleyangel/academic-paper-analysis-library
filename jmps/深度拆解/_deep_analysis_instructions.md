# JMPS 论文深度拆解与素材提取说明

本轮目标：基于 `论文拆解框架与模板.md`，对每篇 JMPS 论文做“细致拆解 + 写作素材提取”。上一轮 `jmps/拆解/papers` 是标准版，本轮输出到 `jmps/深度拆解/papers`，不要覆盖上一轮文件。

## 输入

- 总模板：`论文拆解框架与模板.md`
- PDF 原文：`jmps/文献/*.pdf`
- 已抽取全文：`jmps/文本/txt/*.txt`
- 提取元数据：`jmps/文本/metadata/*.json`
- 上一轮标准拆解可作为辅助参照：`jmps/拆解/papers/*.md`

## 输出

- 每篇一个深度拆解文件：`jmps/深度拆解/papers/{stem}.md`
- 每批一个摘要：`jmps/深度拆解/batch_summaries/batch_{N}.md`

## 深度要求

每篇常规研究论文建议不少于 10000 个中文字符，优先保证信息密度，不要用空话凑长度。短文、Erratum、Correction 可按真实类型简化，但必须说明为什么不能完整套用研究论文模板。

请基于全文阅读，包括 abstract、keywords、introduction、methods/theory/model、results、discussion、conclusion、figure captions、references。图像本体若未直接读取，不要编造视觉细节；应写“需结合 PDF 图像核查”。

## 版权与摘录边界

可以提取短语、句型骨架、术语和表达模式，但不要大段复制原文。常用句式优先改写成“可复用句型骨架”，例如 `Despite advances in X, Y remains unresolved because Z.`。如果确需短引，单条保持很短。
